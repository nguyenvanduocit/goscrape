A field report

An instrument
for the preservation
of the web. A small, console utility — written in Go — that walks a website, fetches what is necessary, and lays it down on disk for later, quieter reading.

The open web is not eternal. Pages disappear behind logins, redirects, paywalls, and quiet sunsets. goscrape is the workshop tool one keeps within arm's reach — a single binary that copies a site, rewrites its links to point inward, and serves it back from localhost:8080 as if nothing had changed.

§01 — Installation

Two paths
to the binary.

Note.
Either route yields the same artifact: a single, statically-linked executable. No daemons. No agents. No telemetry.
i.

From source, via go install

If a recent Go toolchain is at hand. The binary lands in $GOBIN, which is usually ~/go/bin.

Specimen 02 $ go install github.com/cornelk/goscrape@latest
ii.

From a release

Pre-built binaries for macOS, Linux, and Windows are shipped with each tagged release. Unpack and place anywhere on $PATH.

Specimen 03 $ curl -L .../goscrape_darwin_arm64.tar.gz | tar xz
$ mv goscrape /usr/local/bin/
§02 — Usage

A small grammar,
plainly stated.

Caveat lector.
Defaults are deliberately conservative — depth 10, no external assets, polite request pacing.
i.

Mirror a site.

Walks links breadth-first from the root, fetches every asset it touches, and writes a browsable directory to disk.

command $ goscrape https://example.com
ii.

Read it back, offline.

Spins up a tiny static webserver against the downloaded directory — the archive behaves like the original.

command $ goscrape --serve example.com
iii.

Narrow the field.

PERL-compatible regular expressions decide what is fetched and what is left where it is.

command $ goscrape \
  --include "/docs/" \
  --exclude "\\.pdf$" \
  https://example.com
iv.

Save the bytes.

Re-encode JPEG and PNG assets at a lower quality on the way down — the archive ends up dramatically lighter on disk.

command $ goscrape --imagequality 70 \
  https://example.com
§03 — On its merits

Why a new
scraper, exactly?

◆   with apologies to wget, httrack, & Teleport Pro   ◆
a.
Exclusions are honored.

Unlike wget, the URLs you tell it to skip are not silently fetched anyway.

b.
No partial wreckage on disk.

Interrupt a run and the work-in-progress files do not linger as half-formed husks.

c.
Resumable by default.

Assets already downloaded are quietly skipped on the next pass.

d.
External assets, included.

Stylesheets and images served from third-party domains come along.

e.
Image weight, reduced.

An optional re-encode pass on JPEG and PNG keeps archives tidy.

f.
One binary, every platform.

Anywhere Go runs, this runs. No daemons, no runtime, no surprises.

g.
Polite by default.

Sensible defaults — depth, headers, pacing — make the first run safe to try.

h.
Free, and open.

MIT-licensed. Read the source, file a fix, ship a fork.
